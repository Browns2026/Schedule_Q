# 1인 게임 제작 끄적거림

1. '요일별 역할 분담' 또는 '주차별 역할 분담' 활용하기하루에 기획서 쓰다가, 도트 찍다가, 코딩까지 하면 뇌에 과부하가 오고 진도가 안 나갑니다. 모자를 바꿔 쓰듯 역할을 명확히 분리하세요.
   방법 A (요일별 쪼개기): 월~화요일은 프로크리에이트로 아트(도트) 작업, 수~금요일은 게임 엔진으로 유니티/언리얼 코딩, 토요일은 테스트 및 기획 보완, 일요일은 휴식.
   방법 B (주차별 쪼개기 - 추천): 1주 차는 오직 그래픽 리소스만 생산, 2주 차는 그 리소스를 게임에 넣고 기능 구현만 집중.

2. '핵심 재미(Core Loop)'부터 만들기 (1개월 차 목표)스토리, 오프닝 화면, 상점 시스템 같은 부가적인 요소는 일단 전부 뒤로 미루세요.도트 캐릭터가 움직여서, 적을 공격하고, 적이 쓰러진다. 이 핵심 규칙 딱 하나만 작동하는 최소한의 빌드(MVP)를 한 달 안에 만드는 것을 첫 스케줄로 잡아야 합니다. 이게 재미없다면 뒤의 일정은 아무 의미가 없어지므로, **프로토타입** 을 빠르게 뽑는 것이 1인 개발의 최우선 과제입니다.

3. 무조건 일정을 반으로 줄이는 '기획 다이어트'1인 개발자가 일정을 못 지키고 포기하는 가장 큰 이유는 **'만들고 싶은 게 중간에 계속 늘어나서'** 입니다.일정을 짤 때 "이 기능 없어도 게임 플레이가 성립하나?"를 스스로에게 계속 질문하세요.없어도 돌아간다면 과감히 '출시 후 업데이트' 항목으로 빼버려야 완성 단계까지 갈 수 있습니다.

4. 1인 개발 추천 마일스톤 예시 (3달 단기 완성 기준)1~2주 차 (핵심 구현): 프로크리에이트로 주인공/적 기본 도트 그리기 + 이동 및 공격 시스템 코딩3~4주 차 (콘텐츠 확장): 스테이지 1개 분량의 맵 배치 + 적 AI 종류 2~3가지 추가5~8주 차 (시스템 구축): 타이틀 화면, 게임 오버/클리어 루프, 사운드(BGM/효과음) 삽입, 간단한 UI 구성9~12주 차 (폴리싱 및 버그 수정): 무한 테스트, 옵션 창 만들기, 빌드 파일 추출 및 스토어 업로드 준비

5. 혼자서 일정을 지키기 위한 팁 (칸반 보드)컴퓨터 모니터 옆이나 노션(Notion)에 딱 세 칸을 만드세요.해야 할 일 (To Do): 이번 주에 할 일 딱 3~5개만 적기진행 중 (In Progress): 무조건 지금 하는 일 1개만 배치 (멀티태스킹 절대 금지)완료 (Done): 완료된 태스크 쌓아두기 (쌓이는 걸 보며 성취감을 느껴야 번아웃이 안 옵니다)
